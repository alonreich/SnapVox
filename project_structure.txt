SnapVox v3.0 - "The Ghost" Reality Map (Avalonia + ImageSharp Edition)
================================================================================

SYNCHRONIZED BRANCHES MANDATE
----------------------------
CRITICAL: The software is maintained as two functional branches from a SINGLE codebase:
1. Branch 1: Native (Windows OCR Engine).
2. Branch 2: Compiled with Embedded Tesseract OCR Engine (LTS/Offline).

ENGINEERING STANDARD:
Every change (UI, logic, timers, overlay effects, editing tools) MUST be applied to both branches simultaneously.
Agents MUST ensure that features remain 100% synchronized between branches, except for the underlying OCR engine implementation itself.
Both branches utilize the high-performance SixLabors.ImageSharp pipeline for image manipulation and OCR pre-processing.

GOAL & PURPOSE
--------------
SnapVox is a professional-grade, ultra-fast screen capture and OCR tool optimized for .NET 9 Native AOT.
It serves users who need rapid imaging workflows with 100% privacy (zero cloud dependencies).
The application follows a "Ghost" philosophy: it stays out of sight until triggered, performs tasks with sub-millisecond responsiveness, and is strictly dependency-free.

CORE MANDATES (THE "GHOST" RULES)
---------------------------------
1. **Zero UI Friction:** Tray icon click triggers capture immediately. No splash screens.
2. **Zero WinForms/GDI+:** No trace of legacy .NET UI wrappers. Pure Avalonia UI + SixLabors.ImageSharp.
3. **Symbol-Only UI:** The Editor uses standard Windows "Segoe MDL2 Assets" symbols. No image-loading overhead.
4. **Thin-Client Interop:** Use direct Win32/C++ level calls (CsWin32) for performance, bypassing "heavy" .NET wrappers.
5. **Native Finality:** Compiled to machine code via .NET 9 Native AOT. No JIT, no runtime required.
6. **Async Everything (Engine Audit Parity):** All I/O, OCR, and cleanup operations MUST be non-blocking. 
   - No `Thread.Sleep`. Use `Task.Delay`.
   - No `.Result` or `.Wait()`. Use `await` or `IAsyncDisposable` / `DisposeAsync`.
   - Both OCR engines must validate required language packs (EN/HE) before execution.
7. **One-Click Sequential Finality (Zero Prompts):** The editor core is "Zero-Dialog". Only the following three actions trigger the 1.0-second blinking visual overlay before automatically closing the editor window:
   - "Save Image" (Ctrl+S): Saves 100% quality JPG to user's `Downloads` folder AND `%TMP%\SnapVox\` immediately.
   - "Copy to Clipboard" (Ctrl+C): Copies `CF_DIB` to clipboard AND saves 100% quality JPG to `%TMP%\SnapVox\`.
   - "OCR": Copies text to clipboard, saves `.txt` to `%TMP%\SnapVox\`, and opens in Notepad.
   - **Soft-Fail Fallback:** If the `Downloads` folder is missing/unwritable, popup a folder selection dialog ONCE, memorize the choice, and resume zero-prompt behavior for all future saves.
8. **Hardcoded JPG-Only Output:**
   - PNG ingestion is allowed (opening files or injecting from clipboard).
   - ALL output (manual saves, 24-hour backup, automated history) is ALWAYS `.jpg`. 
   - Quality is ALWAYS 100%. No quality dialogs.
9. **Automated 24-Hour Retention:**
   - **Immediate Raw Backup:** EVERY capture (region, full, window) immediately saves a raw 100% quality JPG to `%TMP%\SnapVox\` before any editing begins.
   - All artifacts (images, text) purged from `%TMP%\SnapVox\` after 24 hours.
   - Cleanup task runs on startup and every 4 hours to purge the `%TMP%\SnapVox\` directory of expired files.
10. **Folder Structure Mandates:**
   - Persistent Install: `C:\Program Files\SnapVox\`
   - Persistent Logs: `%PROGRAMDATA%\SnapVox\`
   - User Config: `%AppData%\SnapVox\`
   - Local App Data: `%LocalAppData%\SnapVox\`
   - **Unified Temporary Root:** `%TMP%\SnapVox\`
     - All user-initiated saves, background snapshots, and OCR outputs MUST use this path for 24-hour retention.
     - Subfolder `\Lifecycle\`: Used for installer unpacking and self-cleanup.
     - Subfolder `\Runtime\`: Used for transient DLL extraction.
   - **Installation Log:** Saved EXCLUSIVELY to a standalone file at `%TMP%\SnapVox_Installation.log`.
11. **Visual Notification Overlays:**
   - Any successful download or OCR action MUST trigger a **blinking Red/White/Blue overlay** for 1.0 seconds (4x 250ms cycles).
   - Text variants: "IMAGE SAVED & COPIED" and "TEXT COPIED & SAVED".

TELEMETRY & LOGGING MANDATE (SINGLE SOURCE OF TRUTH)
----------------------------------------------------
1. **Comprehensive User Action Audit:** Secret performance trackers (RAM/CPU/Thread tracing) are strictly forbidden. Instead, the primary application logger MUST capture 100% of user interactions.
2. **Recorded Events:** Screenshot coordinates, keyboard keys pressed, action cancellations, editor initializations, picture saves/downloads, and precise shape/object placements (including X/Y canvas coordinates).
3. **Success/Fail Finality:** Every logged action must culminate in a clear `[SUCCESS]` or `[FAIL]` status. Failures must include explicit, detailed exception data identifying the root cause. This log operates as the absolute single source of truth for all troubleshooting.
4. **Build Logging Tracking:** Compilation execution and output MUST be tracked via a `.\build.log` file at the repository root.

DEPLOYMENT & PACKAGING MANDATES (CRITICAL)
------------------------------------------
1. **Single-File Installer:** The deployment output for EACH branch MUST be a single Native AOT `.exe` installer file.
2. **DLL Persistence:** Any extracted permanent `.dll` dependencies MUST reside exclusively in `C:\Program Files\SnapVox\`.
3. **Unpacked Persistence:** When the single `.exe` installer is run, it MUST unpack its payload into: `C:\Program Files\SnapVox\`

FUNCTIONAL SPECIFICATIONS (THE USER EXPERIENCE)
-----------------------------------------------
1. **Hotkey Triggers:**
   - `PrintScreen`: Starts custom rubberband region selection.
   - `Alt + PrintScreen`: Captures EXACTLY the current active window/app size only.
   - **High-Compatibility Copy:** Every capture is AUTOMATICALLY pushed to the clipboard as raw `CF_DIB` data for seamless pasting.
2. **Visual Feedback & Collision Mechanics:**
   - Systray Icon: Turns **RED** immediately upon entering capture mode. Returns to **BLUE** when finished.
   - **Intelligent Cursor Dodging:** On-screen instruction banners actively calculate bounding-box collisions and jump away from the mouse cursor to ensure the capture area is never obstructed.
3. **Unified Magnifier Engine:**
   - Shared circular UI style for both region selection and color sampler.
   - **Smart-Flip Region Selection:** Lens stays 30px diagonally away from cursor, "flipping" quadrants to avoid covering the capture area. 2x zoom.
   - **Color Sampler Magnifier:** Fixed-center logic with 8x zoom, live hex label, precise red crosshair, and full multi-monitor support.
4. **Magnetic Rubberband Snapping:**
   - Cursor over window triggers a high-visibility **Blue Highlight Border** snap with a strict **10-pixel proximity threshold**.
   - Initiating the rubberband selection directly on a window title area renders a dynamic "ghost effect" preview.
   - Coordinate accuracy is 100% DPI-aware, applying `RenderScaling` to logical Avalonia coordinates.
5. **OCR Painter Mode & Preemptive Caching (The "T" Toggle):** - Hitting `T` during selection toggles "OCR Painter" mode.
   - **Preemptive Background Execution:** Toggling the mode instantly triggers silent OCR across the entire virtual desktop before the region is drawn, guaranteeing zero-latency text extraction upon selection release.
6. **In-Editor OCR:** Clicking OCR symbol extracts text, copies to clipboard, opens Notepad, and saves `.txt` to `%TMP%\SnapVox\`.
7. **Intelligent Editor Display:** Implements parallelSniplets, Cascading Positioning, Majority Monitor Centering, and Adaptive Sizing to ensure non-overlapping, context-aware window placement across multiple displays.
8. **Download Workflow:** Clicking "Download" saves 100% JPG to `%TMP%\SnapVox\`, copies to Clipboard as `CF_DIB`, shows overlay, and closes window sequentially.
9. **Shell "Open With" Integration (Secondary Instance Routing):** Right-clicking any image format in File Explorer and choosing "Open with SnapVox" launches the Editor directly, bypassing Tray Icon initialization. All outputs from this workflow will be strictly saved/converted to `.jpg`.
